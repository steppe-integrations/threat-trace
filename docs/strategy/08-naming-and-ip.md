# 08 — Naming and IP

## Purpose

**"Threat Tracer" is already a Carbon Black / VMware / Broadcom investigation tool** ([source](https://developer.carbonblack.com/2025/06/announcing-threat-tracer-a-new-investigation-tool-for-carbon-black-cloud)). Before any external pitch, public launch, GitHub readme, conference talk, or domain purchase, we need to resolve the brand/IP collision. This is also a chance to choose a name that **supports the broader framework play**, not just the security demo built on it.

## Key questions

- Is "Threat Tracer" / "Threat Trace" trademarked by Carbon Black, Broadcom, or VMware? In which Nice Classification classes (likely 9, 42)?
- Does our use create infringement risk in a different sub-category? (Probably yes — same industry, same buyer.)
- If we abstract the framework beyond security, does the security-flavored name still make sense in AML / radiology / insurance contexts?
- What's the right name for the **framework** (vs the security demo built on it)? Two names — product and framework — may be the right answer.
- What name communicates **rehearsal / experience / traceability** without being cute?

## Considerations

- Product/framework is broader than security — name should travel to medical, AML, insurance, legal verticals
- Memorable + searchable + .com or .ai available
- Must not conflict with: Carbon Black Threat Tracer; existing AI agent products; existing trust/explainability tools (Truera, Fiddler, Arthur, Arize, Credo AI, Holistic AI)
- "Steppe Integrations" is the company brand — products can be branded under it or independently

## Brainstorm (not prescriptive — for review, not selection)

For the **framework**:
- "Mech Suit" — owns the founder's methodology metaphor; broad and memorable; verify trademark
- "Apprentice" — captures the rehearsal/learning frame; may be over-claimed in AI tooling
- "Glassbox" / "Glasswork" — counterpart to "black box"; transparency as the core promise
- "Cogency" / "Cogent" — reasoning + clarity; possibly too abstract
- "Rehearsal" / "Ride-Along" — descriptive, may be too generic

For the **security demo specifically**:
- Keep "threat-trace" as internal codename only; market under a different name
- Or rename outright — "Sentinel Walk," "Watchtower," etc.

For the **audit play** (brief 10):
- "AgentAudit," "Glassbox Audit," "Audit-as-Code," "Witness"

## Decision tree

| Outcome | Action |
|---|---|
| "Threat Tracer" / "Threat Trace" passes USPTO TESS + common-law search | Keep as security product name; still rename framework separately |
| Either fails | Rebrand the demo before any external launch — sooner is cheaper |
| Framework name conflicts with active SaaS in adjacent space | Pick alternate from brainstorm |
| Multiple candidates pass | Test 3 with 5 advisors via 30-second "what do you think this does?" gut-check |

## Starter steps (this week)

1. **USPTO TESS search** for "threat tracer," "threat trace," and top 5 framework brainstorm candidates. Free, takes 30 minutes. Document results.
2. **Domain availability check** — .com, .ai, .io for top 5 candidates via Namecheap or Porkbun. Note prices; .com under $10K is buyable, .ai under $5K is buyable.
3. **Common-law search** via Google + LinkedIn + Crunchbase + GitHub for live products with the same/similar names.
4. **Ask 3 trusted advisors** (mix: 1 founder, 1 buyer, 1 marketer) which name signals what they think we do. 30-second response, no explanation upfront.
5. **Decide and commit by Friday** — naming is a tax that compounds with time. Renaming after 100 GitHub stars or 5 customer mentions is significantly more expensive than now.

## Risks / caveats

- **Cease-and-desist letters from Broadcom would be public reputation damage** — they're a litigious large corp. Even if we'd win on a Nice-class technicality, the email itself is the cost.
- **Renaming late** costs domain authority, GitHub stars, social mentions, content backlinks, and customer trust. Cheap now, expensive later.
- **Bad name = forever**. Spend 1 week max on this; get external feedback; commit. Don't bikeshed indefinitely.
- **".ai" domains are increasingly contested** and sometimes politically coded (the .ai TLD is Anguilla's). Worth checking domain registrant policies if international expansion matters.
- The **Steppe Integrations** brand can absorb most product-naming choices — leverage that. Product doesn't need to carry the company.
- Don't pick a name **just because the .com is available**. Availability is necessary, not sufficient.
